# 截图与演示数据指南

本指南用于补齐 M5 所需的发布截图，并保证 README、GitHub Release、应用市场素材风格统一。

## 1. 目录与命名

截图统一放在 `docs/screenshots/`，统一使用 `webp` 格式，建议使用以下文件名：

- `console-libraries.webp`
- `console-tree.webp`
- `console-editor.webp`
- `console-settings.webp`
- `dashboard-widget.webp`
- `site-index.webp`
- `site-detail.webp`

建议额外保留一份原始大图到本地，不必提交仓库。

## 2. 演示数据

建议至少准备以下 2 个文档库，避免截图内容过空：

### 文档库 A：产品文档

- 名称：`产品文档`
- slug：`product-docs`
- 描述：用于展示典型的产品手册目录
- 封面：使用浅色、低干扰背景图

文档树建议：

- 快速开始
- 安装部署
- 配置说明
- API 概览（其下包含「认证」「错误码」两个子文档）

### 文档库 B：帮助中心

- 名称：`帮助中心`
- slug：`help-center`
- 描述：用于展示文档库首页的多入口布局

文档树建议：

- 常见问题
- 账号管理
- 权限说明

## 3. 拍摄要求

- 使用桌面端宽度，建议浏览器内容区宽度不低于 `1440px`
- 使用同一站点主题、同一套演示数据
- 优先浅色主题和整洁背景
- 截图前清空无关通知、Toast、调试面板和浏览器插件痕迹
- 如果页面包含长文档，滚动到能同时展示结构与正文的位置

## 4. 截图清单

### `console-libraries.webp`

- 页面：后台文档库列表页
- 目标：展示文档库名称、描述、封面和管理入口
- 要求：列表至少出现 2 个文档库

### `console-tree.webp`

- 页面：后台文档树管理页
- 目标：展示树形层级、拖拽区和节点操作
- 要求：至少有 3 层目录关系

### `console-editor.webp`

- 页面：后台文档编辑页
- 目标：展示左侧导航树、右侧 Markdown 编辑器和工具栏
- 要求：正文中包含标题、列表、代码块和站内文档链接

### `console-settings.webp`

- 页面：后台文档设置页
- 目标：展示布局编排或渲染配置表单
- 要求：尽量包含文档库首页布局配置区

### `dashboard-widget.webp`

- 页面：控制台仪表盘
- 目标：展示文档统计小组件和快捷入口

### `site-index.webp`

- 页面：`/docs`
- 目标：展示文档库首页布局、封面和文件夹/文档库入口

### `site-detail.webp`

- 页面：`/docs/{librarySlug}/{docSlug}`
- 目标：展示左侧文档树、正文大纲和正文内容
- 要求：滚动到大纲高亮可见的位置

## 5. 交付顺序

1. 按本指南准备演示数据。
2. 先拍后台 5 张，再拍前台 2 张。
3. 将最终图放入 `docs/screenshots/`。
4. 发布前检查 README 和 Release 说明中的截图引用是否齐全。
